# AIエージェント利用ガイド

このガイドは、HRSチーム開発で **Claude Code / OpenAI Codex / Cursor / GitHub Copilot** などの
AIコーディングエージェントを使うメンバー向けの手引きです。

「AIに任せきり」ではなく、**AIを共同作業者として安全に使う** ことを目的にします。
チーム全員が同じルールで使うことで、成果物の品質と一貫性を保ちます。

## 前提: まず読むもの

1. [README.md](../README.md) … プロジェクト全体像
2. [CONTRIBUTING.md](../CONTRIBUTING.md) … 開発ルール (ブランチ・コミット・PR)
3. [AGENTS.md](../AGENTS.md) … AIエージェント向けの作業規約 (AI自身が読む正本)
4. Gitに不慣れな場合は [git-beginner-guide.md](git-beginner-guide.md)

AIエージェントは作業前に `AGENTS.md` (Claude Code は `CLAUDE.md` 経由) を自動で読みます。
ルールを変えたいときは、このガイドではなく `AGENTS.md` を更新してください。

## ツール別のはじめ方

### Claude Code

```powershell
# インストール (未導入の場合)
npm install -g @anthropic-ai/claude-code

# リポジトリのルートで起動
cd HRS
claude
```

- 起動すると同じディレクトリの `CLAUDE.md` → `AGENTS.md` を読み込みます。
- 最初に「Issue #<番号> の作業をしたい」と伝えると、ルールに沿って進めてくれます。

### OpenAI Codex / その他 AGENTS.md 準拠ツール

- リポジトリルートの `AGENTS.md` を自動で参照します。
- 追加設定は不要です。作業対象の Issue 番号を伝えてください。

### Cursor / GitHub Copilot など

- これらも `AGENTS.md` を参照する設定が可能です。参照しない場合は、
  チャットの冒頭で `AGENTS.md` の内容を読むよう指示してください。

## 推奨ワークフロー

1. GitHub Issues から担当する Issue を1つ選ぶ。
2. AIに **Issue番号と目的** を伝える (例: 「#5 のユースケース記述のドラフトを作って」)。
3. AIにブランチ作成 → 作業 → コミットまでやってもらう。
   - ブランチ名・コミット形式は CONTRIBUTING.md / AGENTS.md のルールに従わせる。
4. **生成物を人間が必ずレビューする** (下記「レビューと責任」)。
5. Pull Request を作成する。
   - **PRの最終送信・レビュー依頼は人間が確認してから**。AIに勝手に送らせない。
6. レビューを受けてマージする。

## レビューと責任

- AIの生成物は **下書き** として扱う。最終的な正しさの責任は人間 (作成者) にある。
- 特に確認すること:
  - UMLの用語・概念がHRSの問題領域に合っているか (実装都合になっていないか)。
  - ユースケース記述・図・コードで用語が一貫しているか。
  - 各 docs サブディレクトリ / `.github/ISSUE_TEMPLATE/review.md` のレビュー項目を満たすか。
- AIが生成したコードや文章を、内容を理解しないままマージしない。

## 禁止・注意事項

- **授業の配布物・提供コード・個人情報・提出条件に関わる非公開情報を、AIへの入力やコミットに含めない。**
  このリポジトリはPublicです (`docs/チーム開発1.pdf` 以外の配布物は追加しない)。
- 技術スタックは未定 (Issue #23)。スタックに依存するコードや設定ファイルを
  AIに独断で生成・追加させない。決定後に着手する。
- AIに `main` へ直接コミット・push させない。必ずブランチ + PR 経由。
- APIキーやトークンなどの秘密情報をプロンプトや `.env` でコミットしない。

## 困ったとき

- ルールが曖昧な場合は、AIに判断させず人間 (チーム) で決める。
- ルール自体を更新したいときは `AGENTS.md` を編集し、PRでチームに共有する。
