# 受入テストチェックリスト

UC 記述に基づく手動確認チェックリスト（#46 での結合確認・デモ予行に使用）。

## 事前準備

```powershell
npm run prisma:seed   # 部屋タイプ・部屋の初期データを投入
npm run dev           # ローカルサーバーを起動 → http://localhost:3000
```

確認結果は「結果」欄に「✓ 合格 / × 不合格 / ― 未実施」で記録する。

---

## UC1: 部屋を予約する

**参照**: `ユースケース記述_部屋を予約する.md`  
**前提**: ローカル DB に部屋タイプ・部屋が seed 済み

### 正常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 1-1 | `/reservations/new` を開く | ステップ 1 の日程入力フォームが表示される | |
| 1-2 | チェックイン・アウト日を未来日、人数を 1〜4 に設定し「次へ」 | 空室一覧が表示される（部屋タイプ名・料金・泊数合計が正しい） | |
| 1-3 | 部屋を選択し「次へ」 | 利用者情報入力画面が表示される | |
| 1-4 | 氏名・メールアドレスを入力し「次へ」 | 予約内容確認画面が表示される（入力内容が正しく反映されている） | |
| 1-5 | 「予約を確定する」をタップ | 予約番号が発行され完了画面が表示される（`HRS-YYYYMMDD-NNNN` 形式） | |
| 1-6 | （メール確認）登録したアドレスに確認メールが届く | 予約番号・日程・料金が記載されている | |

### 異常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 1-E1a | チェックイン日を空欄で「次へ」 | 「日付を選択してください」旨のエラーが表示される | |
| 1-E1b | チェックアウト日をチェックイン日以前に設定し「次へ」 | 「チェックアウト日はチェックイン日より後にしてください」が表示される | |
| 1-E1c | 人数を 0 または 11 で入力し「次へ」 | バリデーションエラーが表示される | |
| 1-E4a | メールアドレスを空欄で「次へ」 | 「メールアドレスを入力してください」が表示される | |
| 1-E4b | 不正な形式（`abc`）のメールアドレスで「次へ」 | 形式エラーが表示される | |

## UC2: 予約を確認する

**参照**: `ユースケース記述_予約を確認する.md`  
**前提**: UC1 で作成した予約が存在する

### 正常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 2-1 | `/reservations/lookup` を開き、予約番号と氏名（姓・名）を入力し「予約を照会する」 | 予約内容（日程・部屋タイプ・ステータス・料金）が正しく表示される | |
| 2-2 | ステータスが `RESERVED`（当日）の場合 | 「チェックインへ進む」ボタンが表示される | |
| 2-3 | ステータスが `RESERVED`（当日以外）の場合 | 「この予約をキャンセルする」ボタンのみ表示される | |
| 2-4 | ステータスが `CHECKED_IN` の場合 | 「チェックアウトへ進む」ボタンが表示される | |

### 異常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 2-E1a | 存在しない予約番号を入力し照会 | 「該当する予約が見つかりませんでした」旨が表示される | |
| 2-E1b | 正しい予約番号に対し氏名を間違えて入力し照会 | 「該当する予約が見つかりませんでした」旨が表示される | |

## UC3: 予約をキャンセルする

**参照**: `ユースケース記述_予約を確認する.md`（キャンセル機能を含む）  
**前提**: `RESERVED` 状態の予約が存在する

### 正常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 3-1 | `/reservations/cancel` を開き、予約番号と氏名を入力し「取消内容を確認する」 | キャンセル料・日程が表示される確認画面が開く | |
| 3-2 | 「キャンセルを確定する」をタップ | 「キャンセルが完了しました」旨が表示される | |
| 3-3 | キャンセル後に同じ予約番号で照会 | ステータスが `CANCELLED` と表示される | |

### 異常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 3-E1 | 存在しない予約番号でキャンセル操作 | 「一致する予約が見つかりませんでした」旨が表示される | |
| 3-E2a | `CANCELLED` 済みの予約番号でキャンセル操作 | 「既にキャンセルされています」旨が表示される | |
| 3-E2b | `CHECKED_IN` の予約番号でキャンセル操作 | 「チェックイン済みの予約はキャンセルできません」旨が表示される | |

## UC4: チェックインする

**参照**: `ユースケース記述_チェックインする.md`  
**前提**: チェックイン日が**本日**の `RESERVED` 予約が存在する（DB の checkInDate を本日に合わせて作成）

### 正常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 4-1 | `/check-in` を開き、予約番号と氏名を入力し「チェックインする」 | 割り当てられた部屋番号とチェックイン完了メッセージが表示される | |
| 4-2 | チェックイン後に予約照会 | ステータスが `CHECKED_IN`、部屋番号が表示される | |

### 異常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 4-E1 | 存在しない予約番号でチェックイン操作 | 「一致する予約が見つかりませんでした」旨が表示される | |
| 4-E2 | チェックイン日が明日以降の予約番号で操作 | 「本日はチェックイン日ではありません」旨が表示される | |
| 4-E3 | 既に `CHECKED_IN` の予約番号で操作 | 「既にチェックイン済みです」旨が表示される | |

## UC5: チェックアウトする

**参照**: `ユースケース記述_チェックアウトする.md`  
**前提**: UC4 でチェックイン済みの宿泊が存在し、部屋番号が分かっている

### 正常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 5-1 | `/check-out` を開き、部屋番号を入力し「料金を確認する」 | 宿泊日数・料金が正しく表示される | |
| 5-2 | 支払い方法を選択し「チェックアウトする」 | チェックアウト完了メッセージが表示される | |
| 5-3 | （メール確認）登録アドレスに領収書メールが届く | 料金・宿泊内容が記載されている | |
| 5-4 | チェックアウト後に予約照会 | ステータスが `CHECKED_OUT` と表示される | |

### 異常系

| # | 操作 | 期待結果 | 結果 |
|---|---|---|---|
| 5-E1 | 存在しない部屋番号でチェックアウト操作 | 「対応する宿泊が見つかりません」旨が表示される | |
| 5-E2 | チェックアウト済みの部屋番号で再操作 | 「既にチェックアウト済みです」旨が表示される | |

## 全体シナリオ（通し確認）

発表デモで使用する操作順序。上記の個別 UC テストが全て合格した後に実施する。

| # | シナリオ | 期待結果 | 結果 |
|---|---|---|---|
| S1 | 予約 → 確認 → キャンセル | キャンセル完了、ステータスが `CANCELLED` | |
| S2 | 予約 → チェックイン → チェックアウト | 部屋番号が発行され、チェックアウト後ステータスが `CHECKED_OUT` | |
| S3 | キャンセル済み予約でチェックインを試みる | チェックイン不可のエラーが表示される | |
| S4 | チェックアウト後の予約番号でキャンセルを試みる | キャンセル不可のエラーが表示される | |
