## クラスと属性

| クラス     | 属性                                                                                                            |
| ---------- | --------------------------------------------------------------------------------------------------------------- |
| 利用者     | 氏名: `String`、メールアドレス: `String`、電話番号: `String [0..1]`                                             |
| 予約       | 予約番号: `String`、チェックイン予定日: `Date`、チェックアウト予定日: `Date`、宿泊人数: `int`、状態: `予約状態` |
| 部屋タイプ | 種別名: `String`、定員: `int`、基本宿泊料: `int`                                                                |
| 部屋       | 部屋番号: `String`                                                                                              |
| 宿泊       | チェックイン日時: `DateTime`、チェックアウト日時: `DateTime [0..1]`                                             |
| 宿泊料金   | 請求金額: `int`、請求日時: `DateTime`                                                                           |
| 支払い     | 支払金額: `int`、支払日時: `DateTime`、支払方法: `String`                                                       |
| 予約状態   | `予約済`、`チェックイン済`、`チェックアウト済`、`取消済`                                                        |

## 関連と多重度

| 関係              | 多重度                       | 意味                                |
| ----------------- | ---------------------------- | ----------------------------------- |
| 利用者 ― 予約     | `利用者 1` ― `予約 0..*`     | 1人の利用者が複数の予約を行える     |
| 予約 ― 部屋タイプ | `予約 0..*` ― `部屋タイプ 1` | 予約時に希望する部屋タイプを1つ指定 |
| 部屋タイプ ― 部屋 | `部屋タイプ 1` ― `部屋 0..*` | 各部屋は1つの部屋タイプに属する     |
| 予約 ― 宿泊       | `予約 1` ― `宿泊 0..1`       | チェックイン前は宿泊が存在しない    |
| 宿泊 ― 部屋       | `宿泊 0..*` ― `部屋 1`       | 宿泊時に具体的な部屋を1つ割り当てる |
| 宿泊 ― 宿泊料金   | `宿泊 1` ― `宿泊料金 0..1`   | チェックアウト時に料金を確定        |
| 宿泊料金 ― 支払い | `宿泊料金 1` ― `支払い 0..*` | 宿泊料金に対する支払いを記録        |

## モデル上の判断

- 単一ホテルを前提とし、`ホテル`クラスは設けない
- 利用者がWebアプリを直接操作するため、`受付係`はドメインモデルに含めない
- 予約時は部屋タイプを指定し、具体的な部屋はチェックイン時に割り当てる
- チェックアウト日時はチェックイン中には存在しないため任意
- 宿泊料金と支払いはチェックアウト時に作成
- 支払方法は現時点では文字列属性として保持
