## 用語表

| 概念 | 説明 |
| --- | --- |
| 利用者 | 部屋を予約し宿泊する人。Webアプリ上で予約・チェックイン・支払いを自身で操作する。 |
| 予約 | 利用者が部屋(タイプ)を確保した約束。予約番号で識別し、状態を持つ。 |
| 予約状態 | 予約のライフサイクル(予約済・チェックイン済・チェックアウト済・取消済)。 |
| 部屋タイプ | 部屋の種別(シングル等)。定員と基本宿泊料を持つ。予約時に指定する。 |
| 部屋 | 個々の客室。チェックイン時に割り当てられる。 |
| 宿泊 | 予約がチェックインされて成立した滞在の実績。 |
| 宿泊料金 | 1回の宿泊に発生する請求金額。 |
| 支払い | 宿泊料金に対する支払い。 |

## 前提・割り切り (レビューで要確認)

- **単一ホテル**を前提とし `ホテル` クラスは設けない。
- **受付係はモデルに含めない**。Webアプリで利用者がセルフで予約・チェックイン・支払いを行う前提のため、受付係はシステムが識別・記録する対象ではなくアクターに留まる。これに伴い共通の親だった抽象クラス `人物` も廃し、`氏名` を `利用者` に統合した（汎化の子が1つになり必然性が失われるため）。担当受付係を記録する要件が出た場合は再検討する。
- **予約は「部屋タイプ」に対して行い、具体的な「部屋」はチェックイン時に割り当てる**（p.12 でチェックイン時に部屋番号を伝達することに対応）。
- 支払方法は属性 `支払方法` で表す（現金・カードなどの区別が重要になれば汎化に変更）。
- 型(String/int/Date)は目安。実装の型はアーキテクチャ設計以降で確定する。
