# ユースケース記述: 予約をキャンセルする

## 概要

| 項目 | 内容 |
| --- | --- |
| ユースケース名 | 予約をキャンセルする |
| 主アクター | 利用者 |
| 関係者 | なし（セルフサービス前提。ホテルスタッフはシステムに直接関与しない） |
| 目的 | 利用者が不要になった予約をキャンセルする |
| 事前条件 | HRSが利用可能であり，利用者がキャンセル可能な予約（予約済みの状態）を保有している |
| 事後条件 | 予約がキャンセル済みとなり，該当部屋タイプの在庫が回復している |
| 失敗時の事後条件 | 予約は変更されず，利用者にキャンセルできない理由が示されている |

## 基本系列

1. 利用者が予約番号と宿泊代表者の氏名（姓・名）を入力する。
2. HRSが入力内容と一致する予約を特定し，予約内容、キャンセルポリシー、キャンセル料を表示する。
3. 利用者が内容を確認し，キャンセルを指示する。
4. HRSが予約をキャンセル済みに更新する。
5. HRSがキャンセル完了とキャンセル料を表示する。

## 代替系列

### A1: 利用者がキャンセルを中止する

3a. 利用者が予約内容を確認し，キャンセルの中止を指示する。  
3b. HRSは予約を変更せず，キャンセル操作を終了する。

## 例外系列

### E1: 入力内容に一致する予約が存在しない

2a. HRSが入力された予約番号と氏名に該当する予約を見つけられない，または氏名が一致しない。  
2b. HRSが入力内容に一致する予約が存在しないことを表示し，再入力を促す。  
2c. ユースケースは基本系列1に戻る。

### E2: 予約が既にキャンセル済みである

2a. HRSが予約が既にキャンセル済みであることを検出する。  
2b. HRSが既にキャンセルされていることを表示する。  
2c. HRSは予約を変更せず，キャンセル操作を終了する。

### E3: 予約がチェックイン済みである

2a. HRSが予約が既にチェックイン済み（チェックアウトを終えた利用済みを含む）であることを検出する。  
2b. HRSがチェックイン済みの予約はキャンセルできないことを表示する。  
2c. HRSは予約を変更せず，キャンセル操作を終了する。

## 補足

- 予約番号だけでなく宿泊代表者の氏名（姓・名）も照合することで，第三者が予約番号を知っていてもキャンセルできないようにしている（本人照合）。予約確認・チェックインの各 UC も同様の方針を採用している。
- キャンセル料は「チェックイン日前日まで無料、チェックイン当日50%、チェックイン予定日後100%」として、予約済みの宿泊料金から算出する。
- 在庫は予約数の集計（予約済み・チェックイン済みが対象）で判定しており，キャンセル済みは集計対象外となるため，キャンセルによって該当部屋タイプの在庫は自動的に回復する。
