# ユースケース記述: 部屋を予約する

## 概要

| 項目 | 内容 |
| --- | --- |
| ユースケース名 | 部屋を予約する |
| 主アクター | 利用者 |
| 関係者 | なし（セルフサービス前提。ホテルスタッフはシステムに直接関与しない） |
| 目的 | 利用者が希望する宿泊条件で空室を探し、予約番号を受け取る |
| 事前条件 | HRSが利用可能であり、予約対象の部屋情報が登録されている |
| 事後条件 | 予約が成立し、HRSに予約情報と予約番号が記録され、利用者に予約確認メールの送信が試行されている |
| 失敗時の事後条件 | 予約情報は作成されず、利用者に予約できない理由が示されている |

## 基本系列

1. 利用者がチェックイン日、チェックアウト日、宿泊人数を入力する。
2. HRSが入力条件の妥当性を確認する。
3. HRSが対象人数で宿泊可能な日別空室数をカレンダーに表示する。
4. HRSが条件に合う空室を部屋タイプ単位で検索する。
5. HRSが予約可能な部屋タイプ候補、定員、空室数、宿泊料金を表示する。
6. 利用者が予約する部屋タイプを選択する。
7. HRSが宿泊代表者情報の入力画面を表示する。
8. 利用者が姓、名、メールアドレス（必須）、電話番号（任意）を入力する。
9. HRSが入力された宿泊代表者情報の妥当性を確認する。
10. HRSが予約内容の確認画面を表示する。
11. 利用者が予約内容を確認し、予約確定を指示する。
12. HRSが選択された部屋タイプの最新の空室状況を再確認する。
13. HRSが宿泊代表者情報と予約情報を登録する。
14. HRSが予約番号を発行する。
15. HRSが予約番号と予約内容を表示する。
16. HRSが利用者のメールアドレスに予約確認メールを送信する。

## 代替系列

### A1: 利用者が検索条件を変更する

5a. 利用者が表示された部屋タイプ候補を選ばず、検索条件の変更を指示する。
5b. HRSが検索条件を変更できる画面を表示する。
5c. 利用者が検索条件を変更する。
5d. ユースケースは基本系列3に戻る。

### A2: 利用者が部屋タイプ選択を修正する

10a. 利用者が予約内容の確認時に部屋タイプ選択の修正を指示する。
10b. HRSが部屋タイプ候補を表示する。
10c. ユースケースは基本系列6に戻る。

### A3: 利用者が宿泊代表者情報を修正する

10a. 利用者が予約内容の確認時に宿泊代表者情報の修正を指示する。
10b. HRSが宿泊代表者情報の入力画面を表示する。
10c. ユースケースは基本系列8に戻る。

### A4: 利用者が予約を取り消す

任意の入力または確認時点で、利用者が予約操作の中止を指示する。  
HRSは予約情報を作成せず、予約操作を終了する。

## 例外系列

### E1: 入力条件が不正である

2a. HRSがチェックイン日、チェックアウト日、宿泊人数のいずれかに不正な値を検出する。
2b. HRSが不正な項目と修正方法を表示する。
2c. ユースケースは基本系列1に戻る。

### E2: 条件に合う空室がない

3a. HRSが条件に合う空室を見つけられない。
3b. HRSが空室がないことを表示し、条件変更を促す。
3c. 利用者が条件変更を指示した場合、利用者が検索条件を変更し、ユースケースは基本系列3に戻る。
3d. 利用者が中止を指示した場合、HRSは予約情報を作成せず、予約操作を終了する。

### E3: 選択した部屋タイプが予約できなくなっている

12a. HRSが選択された部屋タイプを予約できない状態であると検出する。
12b. HRSが予約できない理由を表示する。
12c. HRSが部屋タイプ候補の選択画面に戻す。
12d. ユースケースは基本系列6に戻る。

### E4: 宿泊代表者情報が不正である

9a. HRSが姓、名、メールアドレス、電話番号のいずれかに不正な値を検出する。
9b. HRSが不正な項目と修正方法を表示する。
9c. ユースケースは基本系列8に戻る。

### E5: 予約登録に失敗する

13a. HRSが宿泊代表者情報または予約情報を登録できない。
13b. HRSが予約を完了できないことを表示する。
13c. HRSは予約番号を発行せず、予約操作を終了する。

## 補足

- 空室検索と予約確定は**部屋タイプ単位**で行う。実際の部屋番号はチェックイン時に割り当てる。
- 宿泊代表者は**姓**と**名**を分けて入力し、予約登録時は `姓 名` の形式で保存する。
- 連絡先は**メールアドレス（必須）**と**電話番号（任意）**の2フィールドで入力する。メールアドレスは予約確認メール・リマインダー・領収メールの送信先として使用する。
- 予約確認メール（基本系列16）はメール送信サービスの障害時もメインの予約処理を妨げない非同期送信で行う。送信が失敗した場合でも予約は成立している。
- チェックイン・予約確認・キャンセルでは予約番号と宿泊代表者の氏名（姓・名）による本人照合を行う（詳細は各 UC 参照）。
