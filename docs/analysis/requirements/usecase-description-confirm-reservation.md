# ユースケース記述：予約を確認する

## 項目・内容
- **ユースケース名：** 予約を確認する
- **アクター：** 利用者
- **事前条件：** 予約が完了し、予約番号が発行されている。
- **事後条件：** 利用者が予約内容を確認できている。

## 基本系列
1. 利用者が、予約確認ページ（/reservations/lookup）にアクセスする。
2. システムが、予約番号と宿泊代表者の氏名（姓・名）の入力フォームを表示する。
3. 利用者が、予約番号と宿泊代表者の氏名（姓・名）を入力して「予約を照会する」ボタンを押す。
4. システムが、入力された予約番号および氏名（姓・名）に該当する予約データをデータベースから検索する。
5. システムが、該当する予約詳細（予約番号、客室、チェックイン/アウト日、人数、代表者名、メール、電話番号、合計金額、状態、および部屋番号）を表示する。
6. 利用者が、表示された予約内容を確認する。

## 代替系列
なし

## 例外系列
### 【入力形式が正しくない場合（バリデーションエラー）】
3a-1. 利用者がボタンを押した際、システムが予約番号の形式不正、または氏名（姓・名）の未入力を検出する。
3a-2. システムが、画面に対応するエラーメッセージ（「氏名を入力してください」等）を表示する。
3a-3. 流れは【基本系列 ステップ3】に戻る。

### 【該当する予約が存在しない場合】
4a-1. システムが、入力された条件（予約番号・氏名）に合致する予約データが存在しないことを検出する。
4a-2. システムが、画面にエラーメッセージを表示する。
4a-3. 流れは【基本系列 ステップ3】に戻る。

### 【通信エラーが発生した場合】
4b-1. システムが、データベース等へのアクセス時に通信エラーを検出する。
4b-2. システムが、「通信エラーが発生しました。時間をおいて再度お試しください。」というエラーメッセージを表示する。
4b-3. 流れは【基本系列 ステップ3】に戻る。
