# HRS 画面ワイヤーフレーム仕様

## 概要

- **Claude Design プロジェクト**: [HRS画面ワイヤーフレーム設計](https://claude.ai/design/p/c5a94f59-bc8e-4197-b59d-88011066c76b)
- **対象アクター**: 宿泊客（ゲスト）
- **対象プラットフォーム**: PC（ブラウザ）＋ モバイル
- **フィデリティ**: 中（レイアウトと主要要素を確定。実画像・実コピーは別途）
- ルーティング・API対応の詳細は [`docs/design/ui-routing-design.md`](ui-routing-design.md) を参照

---

## 1. UMLユースケースと画面の対応

| UC | ユースケース | アクター | 画面フロー |
|----|------------|---------|-----------|
| UC1 | 部屋を予約する | 宿泊客 | ① 空室検索 → ② 検索結果一覧 → ③ 客室詳細 → ④ 宿泊者情報入力 → ⑤ 予約確認 → ⑥ 予約完了 |
| UC2 | 予約を確認する | 宿泊客 | ⑦ 予約照会 → 予約詳細（確認・キャンセル導線） |
| UC3 | チェックインする | 宿泊客 | ⑧ 予約照合・客室確認 → チェックイン完了 |
| UC4 | チェックアウトする | 宿泊客 | ⑨ 精算・請求確認 → チェックアウト完了 |

全UC共通でセルフサービス（宿泊客本人が操作）。認証なし、初期実装では予約番号または部屋番号による照合のみ。

## 2. 画面一覧・採用バリアント

ワイヤーフレームは各画面に複数バリアント（A/B/C）を提示している。以下で採用を確定する。

### ① 空室検索（`/reservations/new` 入口）

| バリアント | 概要 |
|-----------|------|
| A | ヒーロー検索：ホテル外観画像＋大型検索パネル |
| B | 横バー＋一覧同居：検索と結果を同一画面 |
| **C ✓** | **ステップ式（カレンダー先行）：日程 → 客室 → 情報 の3ステップ** |

**採用: C**。モバイルでも1画面1タスクで操作しやすく、ステップバーで進捗が明確になるため。

入力項目: チェックイン日 / チェックアウト日（範囲選択） / 人数

### ② 検索結果一覧（`/reservations/new` Step 2）

| バリアント | 概要 |
|-----------|------|
| **A ✓** | **左サイドバー絞り込み＋リスト：料金スライダ・部屋タイプ・設備で絞り込み** |
| B | カードグリッド＋上部チップフィルタ |

**採用: A**。フィルタ項目が多いため左サイドバーが機能的。PCファースト実装に適する。

各行表示項目: 客室写真 / 客室名 / 定員 / 面積 / 設備 / 1泊料金 / 選択ボタン。満室は非活性表示。

### ③ 客室詳細（`/reservations/new` Step 2 詳細）

| バリアント | 概要 |
|-----------|------|
| **A ✓** | **ギャラリー左＋情報右の2カラム：サムネイル・設備・合計料金・予約ボタン** |
| B | 全幅写真＋タブ＋固定予約バー |

**採用: A**。2カラムレイアウトで実装がシンプル。初期実装に適する。

### ④ 宿泊者情報入力（`/reservations/new` Step 3）

| バリアント | 概要 |
|-----------|------|
| **A ✓** | **フォーム＋右に予約サマリ：PC向け2カラム** |
| B | 単一カラム（モバイル兼用） |

**採用: A**（PC）/ **B**（モバイル幅では自動的に縦積みレイアウトに切替）。

必須項目: 氏名 / フリガナ / 電話番号 / メールアドレス。任意: ご要望。

### ⑤ 予約確認（`/reservations/new` Step 4）

| バリアント | 概要 |
|-----------|------|
| **A ✓** | **内容確認＋料金内訳＋支払い方法選択＋確定ボタン** |
| B | アコーディオン要約＋固定合計バー |

**採用: A**。全情報を一覧できる方が確認画面として適切。

支払い方法: クレジットカード / 現地払い（初期実装では記録のみ、外部決済なし）。

### ⑥ 予約完了（`/reservations/new` Step 5）

バリアントなし（1案のみ）。

表示内容: 予約番号（`HRS-YYYYMMDD-NNNN`）/ 予約内容 / 合計金額 / 確認メール案内。
導線: 予約照会（⑦）へのリンク / トップへ戻る。

### ⑦ 予約照会・予約詳細（`/reservations/lookup`）

| バリアント | 概要 |
|-----------|------|
| A | 照会フォーム：予約番号・宿泊客氏名で検索 |
| B | 予約詳細：ステータスチップ・キャンセルポリシー・キャンセル操作 |

A → B の2ステップ構成。照合失敗時は「予約番号か連絡先が正しくありません」のみ表示（区別しない）。

キャンセルポリシー表示: チェックイン日前日まで無料 / チェックイン当日50％ / チェックイン予定日後100％。

### ⑧ チェックイン（`/check-in`、宿泊客）

| ステップ | 内容 |
|---------|------|
| A | 予約番号＋代表者氏名で照合 → 予約情報・割当客室を表示 → 確定 |
| B | チェックイン完了：割当客室 / 氏名 / 宿泊日 / チェックイン時刻 |

### ⑨ チェックアウト（`/check-out`、宿泊客）

| ステップ | 内容 |
|---------|------|
| A | **部屋番号**で照合 → 精算・請求確認：室料 / サービス料 / 支払い方法選択 / 確定 |
| B | チェックアウト完了：請求合計 / 支払い方法 / 退室時刻 / 領収書発行 |

## 3. デザインシステム

### 配色

| 用途 | CSS変数 | 値 |
|------|---------|-----|
| プライマリ | `--accent` | `#4f6d7a`（スチールブルー） |
| プライマリ濃 | `--accent-strong` | `#3a525c` |
| プライマリ淡 | `--accent-soft` | `#dfe7ea` |
| テキスト（主） | `--foreground` | `#3a3a38` |
| テキスト（副） | `--muted` | `#7a7a72` |
| テキスト（補助・プレースホルダ） | `--placeholder` | `#9a9a93` |
| 境界線 | `--border` | `#e6e6e1` |
| 入力欄境界線 | `--border-input` | `#d6d6d0` |
| 背景（キャンバス） | `--background` | `#f6f6f3` |
| 背景（カード） | `--surface` | `#ffffff` |
| ステータスチップ | — | 文字 `#7a7a72` / 背景 `#f0f0ec` / 枠 `#e0e0db` |

ステータス（予約済 / 滞在中 / 精算済）は絵文字・色分けを使わず、中立グレーのチップで表現する。

### タイポグラフィ

初期実装ではシステムフォントスタックを使用（`Arial, "Hiragino Kaku Gothic ProN", "Yu Gothic", Meiryo, sans-serif`）。余裕があれば Google Fonts（Zen Kaku Gothic New）を検討。

### コンポーネント規約

| 要素 | 規約 |
|------|------|
| カード | 白背景 / `1px solid var(--border)` / `border-radius: 6px` / 薄いシャドウ |
| プライマリボタン (`.btn-primary`) | `--accent` 背景 / 白文字 / `border-radius: 5px` |
| セカンダリボタン (`.btn-secondary`) | 白背景 / `1px solid var(--border-input)` |
| 入力欄 (`.field-input`) | `#fafaf8` 背景 / `1px solid var(--border-input)` / `border-radius: 4px` / 高さ40px |
| ステップバー (`.step-rail`) | 円番号＋ラベル連結。完了=グレー塗り、現在地=プライマリ塗り |
| 写真プレースホルダ | 斜めストライプ＋用途ラベル（実画像差し替え前提） |
| エラー表示 (`.error-box`) | `#fef2f2` 背景 / `#fca5a5` 枠 / `#dc2626` テキスト |

共通CSSクラスは `src/app/globals.css` に定義済み。

### ナビゲーション

- **PC**: ヘッダ固定（HRSロゴ・空室検索・予約照会）。予約フロー内はステップバーで進捗を明示。
- **モバイル**: 1画面1タスクの縦積み。予約サマリは折りたたみ。
- **チェックイン・チェックアウト**: セルフサービス。全画面で統一したトーンを使用。

## 4. データ・命名規約

| 項目 | 規約 | 例 |
|------|------|-----|
| 予約番号 | `HRS-YYYYMMDD-NNNN` | `HRS-20260710-0042` |
| 客室番号 | 階＋連番 | `0805号室`（8階） |
| 金額表記 | `¥` ＋3桁区切り、税込合計を明示 | `¥50,400（税込）` |
| 予約状態 | 予約済（未到着）/ 滞在中 / 精算済 | チップで表示 |

## 5. 未確定事項

| # | 項目 | 現状の仮置き |
|---|------|------------|
| 1 | 用語統一（「部屋」vs「客室」、宿泊客の保持属性） | 画面は「客室」、コードは `roomType` |
| 2 | 料金計算ルール（サービス料率・税・追加課金） | サービス料5%、税込合計を提示 |
| 3 | キャンセル手数料の確定ルールと境界条件 | チェックイン日前日まで無料 / チェックイン当日50% / チェックイン予定日後100% |
| 4 | 客室在庫・割当ロジック（オーバーブッキング可否） | DBトランザクションで先勝ちとする |
| 5 | 認証・会員制の有無（予約照会の本人確認方法） | 初期実装は予約番号＋氏名（姓・名）による照合のみ |
| 6 | チェックイン可能時刻・チェックアウト時刻の制約 | 実装時に確認（`NOT_CHECKIN_DATE` エラーコード対応済） |
