# コラボレーション図: チェックアウトする

- 対象ユースケース: チェックアウトする（#7）
- 対象Issue: #11

## 関与クラス（ロバストネス分析）

| 分類 | クラス | 役割 |
| --- | --- | --- |
| «boundary» | チェックアウトフォーム | 部屋番号の入力と，料金確認後の支払い／中止の指示を受け付ける |
| «boundary» | チェックアウト結果画面 | 宿泊料金・チェックアウト完了・エラーを表示する |
| «control» | チェックアウトコントロール | 宿泊の特定，料金計算，支払い処理，チェックアウト記録を制御する |
| «entity» | 部屋 | 部屋番号で実部屋を特定する |
| «entity» | 宿泊 | 部屋に対応するチェックイン済みの宿泊を特定し，チェックアウト日時を記録する |
| «entity» | 部屋タイプ | 宿泊した部屋の部屋タイプから基本宿泊料を取得する |
| «entity» | 宿泊料金 | 宿泊日数と基本宿泊料から金額を計算する |
| «entity» | 支払い | 宿泊料金に対する支払いを処理・記録する |
| «entity» | 予約 | 宿泊に対応する予約の状態を確認し，チェックアウト済へ更新する |

コントロールはユースケース当たり1個（5個以内）に抑えている．

## メッセージとユースケース記述の対応

| No. | メッセージ | 基本系列 |
| --- | --- | --- |
| 1 | 部屋番号を入力する | 1 |
| 2 | チェックアウトを要求する | 1 |
| 3 | 部屋を特定する | 2 |
| 4 | 宿泊を特定する | 2 |
| 5 | 基本宿泊料を取得する | 3 |
| 6 | 料金を計算する（«create»） | 3 |
| 7 | 料金を表示する | 3 |
| 8 | 支払いを指示する | 4 |
| 9 | 支払いを要求する | 4 |
| 10 | 支払いを処理する（«create»） | 5 |
| 11 | チェックアウト日時を記録する | 6 |
| 12 | 状態をチェックアウト済へ更新する | 6 |
| 13 | 完了と支払い内容を表示する | 7 |

## エラー表示（メッセージ14）と例外系列の対応

メッセージ14「エラーを表示する」は，チェックアウトコントロールからチェックアウト結果画面への表示であり，ユースケース記述の以下の例外系列に対応する．

- E1: 部屋番号に対応する宿泊が存在しない（メッセージ4で特定に失敗）
- E2: 既にチェックアウト済みである（メッセージ4で対応する宿泊／予約が既にチェックアウト済み）
- E3: 支払いに失敗する（メッセージ10で支払い処理に失敗）

## 支払いの中止（メッセージ15）と代替系列の対応

メッセージ15「支払いを中止する」は，料金確認後に利用者がチェックアウトフォームへ中止を指示し，チェックアウトコントロールが支払い・記録を行わずに操作を終了する流れであり，ユースケース記述の代替系列 A1（利用者が支払いを取りやめる）に対応する．
