## 作業時の要点

- Public リポジトリに秘密情報や非公開情報を追加しない
- 1 Issue / 1 成果物 / 1 ブランチ / 1 Pull Request を守る
- `main` に直接コミットまたは push しない
- Issue の範囲外を変更しない
- Pull Request の最終送信とレビュー依頼は人間の確認後に行う
